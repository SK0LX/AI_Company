# AI IT Company 🏢🤖

Мини-IT-компания из AI-агентов, доступная через Telegram-бота. Вы ставите задачу
в чат — **CEO** (управляющий агент) составляет план, спрашивает ваше одобрение и
раздаёт работу команде специалистов, которые пишут **настоящие файлы проекта**,
а при необходимости **реально запускают** установку, сборку и тесты (с вашего
подтверждения на каждую команду).

Команда работает внутри на английском (так модели стабильнее следуют
инструкциям), а с вами CEO общается **по-русски**.

**v2 (платформа агентов):** агенты теперь хранятся в БД и управляются через
**веб-админку** — можно добавлять/менять агентов, их промпты, права и модели без
правки кода. У каждого агента может быть **свой Telegram-бот** для личного
общения. Подробное ТЗ развития — в [`docs/SPEC_v2.md`](docs/SPEC_v2.md).

## Команда

Это стартовый набор ролей (засеивается в БД при первом запуске). Дальше его
можно править и расширять в админ-панели — `slug` остаётся ключом для делегаций.

| Роль | Ключ | Чем занимается |
|------|------|----------------|
| 👔 CEO | `ceo` | Единая точка общения с вами: план, уточнения, делегирование, синтез ответа |
| 📋 Бизнес-аналитик | `business_analyst` | Цели, пользователи, user stories, критерии приёмки, объём |
| 🧩 Системный аналитик | `system_analyst` | Архитектура, требования, модель данных, API-контракты, стек |
| ⚙️ Backend-разработчик | `developer` | Серверный код, API, БД; может запускать код/команды |
| 🎨 Frontend-разработчик | `frontend` | UI, компоненты, интеграция с API |
| 🧪 QA-тестировщик | `tester` | Стратегия и кейсы тестов; **реально прогоняет** pytest/npm test |
| ✏️ UX/UI-дизайнер | `designer` | Пользовательские потоки, вайрфреймы, дизайн-система, доступность |
| 🔍 Ревьюер бэкенда | `backend_reviewer` | Читает реальный backend-код, ловит баги/уязвимости, чинит мелкое |
| 🔍 Ревьюер фронтенда | `frontend_reviewer` | То же для фронтенда (state/props, доступность, баги рендера) |
| 🧭 Тех-лид | `reviewer` | Сверяет файлы с согласованной структурой, переносит/чистит лишнее |

## Возможности

- **План с подтверждением.** Для нетривиальных задач CEO сначала показывает план
  и кнопки **✅ Делаем / ✏️ Изменить / ✖️ Отмена** — работа стартует только после
  одобрения.
- **Один проект — одна папка.** CEO фиксирует единое дерево файлов
  (`backend/`, `frontend/`, `docs/`, …), и все специалисты пишут строго туда.
- **Живой прогресс.** Шаги команды («подключаю такого-то — делаю то-то»)
  копятся в одном редактируемом сообщении; результат специалиста приходит
  отдельно; в конце — явное «✅ Готово».
- **Реальное исполнение (с одобрением).** Агенты могут запускать `npm install`,
  `pytest`, `docker compose up --build` и т.п. — каждая команда показывается вам
  с кнопками **✅ Выполнить / ✖️ Пропустить**.
- **Код-ревью и проверка структуры** перед завершением.
- **Долгая память (Obsidian-вики).** После задачи CEO пишет карточку проекта в
  Markdown-хранилище, которое можно открыть как Obsidian-vault; перед новой
  задачей он подтягивает релевантные заметки.
- **Память диалога** на SQLite — переживает перезапуск (по чату).
- **Общение во время работы.** Пока команда работает, можно писать CEO —
  он ответит и подхватит ваши дополнения в текущую задачу, не начиная новую.
- **Учёт квоты.** Бот оценивает остаток бесплатных запросов OpenRouter и
  предупреждает заранее.
- **Админ-панель (v2).** Веб-страница для CRUD агентов: имя, роль, промпт,
  модель, права (чекбоксы), обязанности, Telegram-токен. Правки применяются к
  работающему боту без рестарта (реестр перечитывается на каждый запрос).
- **Личные боты (v2).** У агента с заданным токеном поднимается собственный
  Telegram-бот — можно написать ему в личку и поговорить именно с этой ролью.
  Добавление/удаление токена подхватывается без рестарта (hot-add).
- **Права рулят инструментами.** Доступ агента к файлам/shell определяется его
  правами (`can_edit_files` / `can_run_shell`), а не захардкоженными списками.

## Архитектура

- **LangGraph** — кастомный супервизор: CEO на каждом шаге возвращает
  структурированное решение (`propose_plan` / `delegate` / `clarify` / `final`),
  а не «передаёт управление» через tool-handoff (устойчивее на разных моделях).
- **Специалисты** — ReAct-агенты с файловыми инструментами
  (`write_file` / `read_file` / `list_files` / `move_file` / `delete_file`) и
  `run_shell` / `run_python`.
- **Провайдеры LLM:** OpenRouter (бесплатные модели по умолчанию), Google Gemini,
  Anthropic — переключаются через `LLM_PROVIDER`.
- **Реестр агентов (v2):** SQLModel-таблицы в `data/app.sqlite` — единый источник
  правды по агентам; `prompts.py` остался источником сидов.
- **Совместная работа (v2 этап 4):** задачи и их таймлайн, делегирование с
  согласия, взаимопомощь и аудит — через `MessageBus` (`src/bus.py`) и
  `src/collab.py`. Права энфорсятся на инструментах (`@requires`).
- **Навыки (v2 этап 5):** переиспользуемые умения в `agents/<slug>/skills/<name>/`
  (`skill.yaml` + `impl.py`); публикуются и перенимаются другими агентами,
  становясь у них вызываемыми инструментами.
- **Доска задач (v2 этап 6):** во вкладке «Доска» админки — kanban по статусам,
  таймлайн событий задачи, граф делегирований между агентами; обновляется в
  реальном времени по WebSocket (`/ws/events`). API: `/api/tasks`, `/api/graph`.
- **Лента активности:** вкладка «Активность» с фильтрами **Все / Задачи /
  Сознания / Система** — единый live-поток действий агентов. «Сознания» — поток
  рассуждений CEO (почему делегирует), «Система» — аудит (права, навыки).
  API: `/api/activity?category=...`.
- **Общая память (Obsidian):** все агенты читают и пишут в ОДНУ базу знаний
  (`WIKI_DIR`) через инструменты `search_memory` / `read_memory` / `list_memory`
  / `save_memory`. Перед работой агент ищет, что команда уже знает; по ходу
  сохраняет «как это работает / решения / карту файлов / как запускать».
  После задачи CEO обновляет структурированную заметку проекта.
- **Проактив (v2 этап 7):** агенты сами пишут в командный чат по событиям
  (задача готова, нужна помощь, отказ) — шаблонными сообщениями, с guardrails
  (cooldown, rate-limit, дедуп, mute, право `proactive` на агента). Включается
  через `ENABLE_PROACTIVE` + `TEAM_CHAT_ID`.

```
src/
├── bot/
│   ├── telegram_bot.py   # Командный бот: сообщения, кнопки, прогресс
│   └── manager.py        # Мульти-бот: команда + личный бот на агента (v2)
├── graph/team_graph.py   # Оркестрация команды (LangGraph), узлы CEO/специалистов
├── agents/
│   ├── prompts.py        # Системные промпты ролей (сиды для БД)
│   └── tools.py          # Файловые инструменты, run_shell, run_python, вики, @requires
├── db/                   # SQLModel: модели агентов/задач/навыков + движок (v2)
├── registry.py           # Реестр агентов: чтение/запись/кэш (v2)
├── bus.py                # Внутренняя async-шина сообщений между агентами (v2)
├── collab.py             # Задачи, делегирование-с-согласия, помощь, чтения доски (v2)
├── events.py             # Live-хаб событий для доски (WebSocket, v2)
├── proactive.py          # Проактивные посты агентов в чат + guardrails (v2)
├── agent_fs.py           # Стандарт папок агентов agents/<slug>/ (v2)
├── skills.py             # Контракт навыка + загрузчик (v2)
├── skill_registry.py     # Каталог навыков, adoption, semver (v2)
├── web/                  # FastAPI: бот + админ-API + страница (один процесс, v2)
├── crypto.py             # Шифрование Telegram-токенов (Fernet, v2)
├── approvals.py          # Мост подтверждения команд (human-in-the-loop)
├── quota.py              # Учёт дневной квоты OpenRouter
└── config.py             # Настройки (pydantic-settings, читает .env)
agents/<slug>/skills/     # Навыки агентов (отслеживаются в git)
main.py                   # Точка входа: бот + админка в одном процессе (polling)
run_admin.py              # Алиас на main.py
tests/                    # Тесты без pytest/LLM: python tests/test_*.py
docs/SPEC_v2.md           # ТЗ развития платформы
```

## Установка и запуск

Требуется **Python 3.11+** (разрабатывалось на 3.13).

```bash
# 1. Клонировать и создать окружение
git clone https://github.com/SK0LX/AI_Company.git
cd AI_Company
python3 -m venv .venv
source .venv/bin/activate

# 2. Зависимости
pip install -r requirements.txt

# 3. Настроить ключи
cp .env.example .env
#   затем впишите в .env свои ключи (см. ниже)

# 4. Запустить платформу (боты + админ-панель в одном процессе)
python main.py             # → http://127.0.0.1:8100
```

Создайте бота у [@BotFather](https://t.me/BotFather) и получите токен; ключ
OpenRouter — на [openrouter.ai/keys](https://openrouter.ai/keys).

### Админ-панель и личные боты (v2)

`python main.py` поднимает боты и одновременно открывает страницу управления
агентами на `http://127.0.0.1:8100` (только localhost). Там можно
добавлять/менять агентов, их промпты, права и модели — изменения подхватывает
работающий бот. (`run_admin.py` оставлен как алиас на `main.py`.)

Чтобы дать агенту **личного Telegram-бота**: создайте отдельного бота у
@BotFather, вставьте его токен в карточку агента в панели — его личный бот
поднимется **сразу** (один процесс, без рестарта). Напишите ему в личку —
ответит в роли этого агента.

## Конфигурация (`.env`)

| Переменная | Назначение |
|------------|-----------|
| `LLM_PROVIDER` | `openrouter` (по умолч.), `google` или `anthropic` |
| `OPENROUTER_API_KEY` | Ключ OpenRouter |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота |
| `AGENT_MODEL` / `CEO_MODEL` | Модели специалистов / CEO (пусто = дефолт провайдера) |
| `WORKSPACE_DIR` | Папка, куда команда пишет проекты |
| `WIKI_DIR` | Папка-vault долгой памяти (открывается в Obsidian) |
| `SHOW_TEAM_CHATTER` / `TRANSLATE_CHATTER` | Показывать прогресс / переводить его на русский |
| `ENABLE_SHELL_EXECUTION` | Разрешить запуск реальных команд (с подтверждением) |
| `ENABLE_CODE_EXECUTION` | Разрешить запуск Python-кода разработчиком |
| `FREE_DAILY_LIMIT` | Дневной лимит бесплатных запросов (50, или 1000 при $10 на OpenRouter) |
| `APP_SECRET` | Ключ шифрования Telegram-токенов агентов (пусто = автогенерация в `data/secret.key`) |

Полный список с комментариями — в [`.env.example`](.env.example).

> **Бесплатные модели OpenRouter** делят **один** дневной лимит на аккаунт
> (50/день при балансе < $10, 1000/день после пополнения на $10 — `:free`-модели
> при этом остаются бесплатными). Многоагентный прогон тратит десятки запросов,
> так что лимит легко исчерпать — бот предупредит заранее.

## Команды бота

| Команда | Действие |
|---------|----------|
| `/start`, `/help` | Справка |
| `/team <задача>` | Поставить задачу всей команде (удобно в группах) |
| `/ask <роль> <вопрос>` | Спросить одного специалиста (`ba`, `sa`, `dev`, `fe`, `qa`, `ux`, `br`, `fr`, `lead`) |
| `/roles` | Список команды |
| `/reset` | Начать диалог с чистого листа |

В личке достаточно просто написать задачу. В группе — упомяните бота или
ответьте на его сообщение. Можно адресовать специалисту префиксом: `dev: …`.

## ⚠️ Безопасность

- **Секреты только в `.env`** (он в `.gitignore` и в репозиторий не попадает).
- **Telegram-токены агентов шифруются в БД** (Fernet, ключ — `APP_SECRET` или
  автоген в `data/secret.key`); через админ-API токен не отдаётся. Сама панель
  пока без авторизации — слушает только `127.0.0.1`; перед публикацией добавьте
  логин.
- **`ENABLE_SHELL_EXECUTION` запускает команды на вашей машине.** Это не
  полноценный сэндбокс — ограничена только рабочая директория проекта; защита в
  том, что **каждая команда требует вашего подтверждения** кнопкой. Включайте
  только на доверенной машине; для продакшена выносите исполнение в
  изолированное окружение (Docker/VM).
